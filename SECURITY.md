# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 0.x.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in ShadeTripTxt, please report it
responsibly:

1. **Do NOT open a public issue.**
2. Email the maintainer or use [GitHub Security Advisories](https://github.com/jrodriguezgar/ShadeTripTxt/security/advisories/new).
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will acknowledge receipt within 48 hours and provide a timeline for a fix.

## Scope

This policy applies to the shadetriptxt Python package and its submodules:
- 	ext_parser — text parsing and normalization
- 	ext_matcher — fuzzy matching
- 	ext_anonymizer — PII anonymization
- 	ext_dummy — fake data generation
- utils — shared utilities
